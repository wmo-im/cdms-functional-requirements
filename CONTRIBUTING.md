# About contributing to Climate Data Management System (CDMS) Functional and Non-functional Requirements

Anyone can contribute to the docs in the public repository:
- <https://github.com/wmo-im/cdms-functional-requirements>

## Issues

[Issues](https://github.com/wmo-im/cdms-functional-requirements/issues) are used to track tasks that contributors can help with.

If you've found something in the documentation content that should be updated, search the open issues to see if someone else has reported the same thing. If it's something new, [open a new issue issue](https://github.com/wmo-im/cdms-functional-requirements/issues/new). We'll use the issue to have a conversation about the problem you'd like to be fixed.

## Pull requests

A [pull request](https://github.com/wmo-im/cdms-functional-requirements/pulls) is a way to suggest changes in our repository. When we merge those changes, they're are incorporated into the PDF version of the document.

To edit a file and create a pull request, [navigate to the directory for the chapter](https://github.com/wmo-im/cdms-functional-requirements/tree/main/standard/sections) that you want to edit and click on the `index.adoc` file for that chapter.

Just above the text of the chapter begins there is a grey bar with options like "Preview", "Code" and "Blame". On the right hand side of this bar click the pen icon to "Edit this file".

Make the changes that you would like to propose and click the green "Commit changes..." button. Type a brief summary of the changes that you are proposing and click the green "Propose changes" button.

### Reviewing your own pull requests

You should always review your own pull request first, before marking it as ready for review by others.

For content changes, make sure that you:

* Confirm that the changes meet the user experience
* Review the content for technical accuracy.
* Check your changes for grammar, spelling, and adherence to [WMO style guide](https://community.wmo.int/sites/default/files/WMO_Style_Guide_2014_en.pdf)
* Check the preview of any pages you have changed. A preview is automatically generated after you submit a pull request and links are added to the pull request. The preview sometimes takes several minutes before it is ready to view. Confirm that everything is rendering as expected. Checking the preview will help you identify problems such as typos, content that doesn't follow the style guide, or content that isn't rendering due to versioning problems. Make sure to check the rendered output for lists and tables, which can sometimes have problems that are difficult to identify in the asciidoc.
* If there are any failing checks in your pull request, troubleshoot them until they're all passing.

=== Requirements Class: Climate Products

[cols="1,4"]
|===
|Requirements class|/req/climate-products
|Target type|Software system
|Dependency|/req/core, /req/quality-control, /req/climate-computation
|===

==== Overview

Support the computation and management of standard climate products from quality-controlled observation
series, including climate normals, standard WMO observation products, and climate indices. This requirements
class draws on the summary statistics computed by /req/climate-computation. Derived products must be clearly
distinguished from raw observations with full documentation of their derivation and data lineage.

==== Requirements

[[req_climate_products_normals]]
===== Climate Normals

[cols="1,4"]
|===
2+^|**Requirement {counter:req-id}**
|Identifier|/req/climate-products/normals
|Statement a|The system SHALL support computation and storage of climate normals in accordance with
WMO guidelines (WMO-No. 1203), including:

* Standard 30-year averaging periods as defined by WMO
* Monthly and annual normals for each observed element
* Recording of the reference period for each set of normals
* Ability to compute and store normals for multiple reference periods simultaneously
|===

[[req_climate_products_wmo_products]]
===== WMO Standard Observation Products

[cols="1,4"]
|===
2+^|**Requirement {counter:req-id}**
|Identifier|/req/climate-products/wmo-products
|Statement a|The system SHALL support generation of the following WMO standard observation products from quality-controlled observation series:

* **World Weather Records (WWR)**: compilation of long-period station records in accordance with WMO-No. 1186, comprising monthly means, extremes, and totals for each observed element over the submission period
* **CLIMAT messages** (FM 71): monthly climatological summaries for submission via WIS 2.0 or other designated exchange mechanisms
* **DAYCLI messages**: daily temperature and precipitation values for submission via WIS 2.0, including required quality flag status and siting classification metadata

|===

[NOTE]
====
This requirement covers the generation of these products from quality-controlled observation
series held in the CDMS. Ingestion of incoming CLIMAT and DAYCLI messages from external sources
is addressed in /req/data-ingestion/formats.

For DAYCLI, generated messages must include siting classification, measurement quality
classification, temperature computation method, and quality control flags for each daily value.
====

[[rec_climate_products_indices]]
===== Climate Indices

[cols="1,4"]
|===
2+^|**Recommendation {counter:rec-id}**
|Identifier|/rec/climate-products/indices
|Statement a|The system SHOULD support computation of standard climate indices, including:

* Temperature and precipitation extremes indices for climate change monitoring
* Standardized Precipitation Index (SPI) and related drought indices
* User-configurable index definitions to accommodate national or regional requirements
|===

[[req_climate_products_management]]
===== Product Management

[cols="1,4"]
|===
2+^|**Requirement {counter:req-id}**
|Identifier|/req/climate-products/management
|Statement a|The system SHALL manage derived products such that:

* Products are versioned, with earlier versions retained and accessible
* Recomputation following correction of source observations generates a new version without
  overwriting prior versions
* Product metadata records the source data period, computation date, and quality constraints applied
* Products are accessible through the data API defined in /req/data-api
|===

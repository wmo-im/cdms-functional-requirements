== Conformance

This specification defines the following conformance classes corresponding to the functional capabilities
of a Climate Data Management System:

- **Core Data Management** (Required): Foundational observation storage and retrieval, provenance metadata management,
  and basic station and sensor metadata storage. This class provides the data management foundation on which all other
  conformance classes depend.
- **Quality Control** (Required): Climate-specific QC procedures including range, temporal, spatial, and climatological checks
- **Data Ingestion** (Required): Multi-format data import, validation, and normalization
- **Data API** (Required): Standards-based data query, retrieval, and distribution interfaces
- **Homogenization** (Recommended): Statistical detection and adjustment of inhomogeneities in climate time series
- **Data Rescue** (Optional): Digitization workflow for historical paper records including imaging and data entry
- **Metadata Management** (Required): Advanced station and element metadata management including WIGOS compliance,
  complete station history, discovery metadata, and metadata quality validation
- **User Interface** (Required): Interactive visualization, data review, and quality control validation capabilities
- **Climate Products** (Recommended): Production and publication of standard climate products and indices
- **Processing Automation** (Recommended): Scheduled and automated execution of routine data processing workflows


[NOTE]
====
The classification of conformance classes as Requirement, Recommendation and Permission aligns with the
Required, Recommended and Optional classifications respectively from WMO-No. 1131.
Required classes must be implemented for a system to be considered
conformant. Recommended classes should be implemented where operationally feasible. Optional classes
may be implemented based on specific organizational needs.
====
